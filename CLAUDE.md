# CLAUDE.md — api.broiler.dev

Express 5 / TypeScript / PostgreSQL / Redis API serving the `3roiler/web` SPA at
broiler.dev. Covers user auth (local + GitHub/Twitch OAuth), a blog with
visibility rules, the Streamclips Germany rating platform (clips, comments,
reports, "Für dich" feed), a 3D-printer workflow (printers, G-code, STL,
print-requests, agent API), DigitalOcean metrics proxy, server-rendered Open
Graph for social crawlers, dynamic `sitemap.xml`, and a DSGVO data-export
endpoint. All HTTP routes live under `API_PREFIX` (default `/api`).

## Commands

- `npm run dev` — tsx watch on `src/app.ts` (needs a running PG + Redis;
  start them via `make pg-start` plus `docker compose up redis -d`, or just
  `make start` which boots the whole stack + runs migrations).
- `npm run build` — `tsc` to `./dist`.
- `npm start` — `node dist/app.js` (production entry, what the container runs
  after `docker-entrypoint.sh` applies migrations).
- `npm run lint` / `npm run lint:fix` — `eslint . --ext .ts`.
- `npm run migrate:up` / `migrate:down` / `migrate:redo` — node-pg-migrate
  against `./migrations`, reads `DATABASE_URL` from `.env` via `dotenv-cli`.
- `npm run migrate:create -- <name>` (or `make migrate-create name=<name>`)
  — scaffolds a new JS migration file in `./migrations`.
- `make start` — `docker compose up -d` + `make migrate-up` (full stack).
- `make run` — same but foreground; `make stop` to tear down.
- `make pg-start` / `pg-stop` / `pg-logs` / `pg-shell` — postgres + pgAdmin
  helpers (pgAdmin is gated behind the `dev` compose profile).
- `make build` — `docker compose build` (image build, not `tsc`).

## Architecture

Layering (per repo README): **Routes -> Controller -> Service -> Config**.
A service may import another service; a service must NOT import a controller.

```
src/
  app.ts             # Express bootstrap, middleware stack, listen()
  routes/            # Express Router composition; auth/permission gates here
  controllers/       # Request parsing, validation, response shaping
  services/          # Business logic + persistence (`persistence.ts` is the
                     # only direct pg/redis client). Exported via index.ts.
  middleware/        # csrf.ts, requirePermission.ts
  models/index.ts    # Plain TS interfaces (no ORM)
migrations/          # node-pg-migrate JS files, prefixed by ms-timestamp
```

**Middleware order in `src/app.ts`** (do not reshuffle without re-reading the
inline comments — order is load-bearing for security):

1. `app.set('trust proxy', 1)` (single proxy hop = Caddy in prod).
2. `cookieParser()`.
3. `express.json({ limit: '64kb' })` + `urlencoded({ limit: '32kb' })`.
   Large bodies go through dedicated `express.raw()` routes.
4. CORS with a strict origin function — credentials enabled, so wildcard is
   forbidden. Prod: `CORS_ORIGIN` allowlist. Dev fallback: localhost/LAN regex.
5. Per-scope rate limiters via an `exactPath()` helper (login 5/15min,
   OAuth 10/15min, OAuth-state 20/15min, twitch-stream 30/min, DSGVO export
   5/hour). Then the global limiter (100/10min), which `skip`s
   `/admin/metrics/*` because the live dashboard polls 7 endpoints.
6. `csrfGuard` (`middleware/csrf.ts`) — placed AFTER rate-limiters
   intentionally so an attacker cannot trigger the auth check unrate-limited.
7. `logger`, then `routes` mounted at `config.prefix`.
8. 404 handler -> `system.errorHandler` (last).

**Auth flow** (`services/system.ts` + `services/auth.ts`)
- JWTs are HS256 via `jose`, signed with `JWT_SECRET` (random fallback in dev),
  carry `sub`, `iss = config.url`, `jti` (uuid), default expiry 24h.
- Token source: `Authorization: Bearer ...` OR `access_token` HttpOnly cookie
  (`sameSite: 'lax'`, `domain = config.cookieDomain`, `path = config.prefix`).
- `authHandler` sets `req.userId` + `req.authSource`. `optionalAuthHandler` does
  the same but never rejects.
- Logout (and `nukeMePlease`) writes the `jti` to Redis under
  `revoked_token:<jti>` with TTL = remaining token lifetime; `verifyToken`
  checks the cache.
- OAuth (Twitch + GitHub) uses a state cookie generated by
  `services/oauth-state.ts` (`oauthStateHandler` factory + timing-safe
  `verifyAndClearOAuthStateCookie`). Cookie is HttpOnly, 10-min TTL.

**Permissions** — `middleware/requirePermission.ts` runs after `authHandler`,
loads from `user_permission` + `group_permission`, caches into
`res.locals.permissions`. Routes compose `[system.authHandler,
requirePermission('foo.bar')]` arrays (see `routes/admin.ts` for the pattern).

**Redis usage** — auth revocation list, OAuth tokens never touch Redis, the
DigitalOcean metrics proxy caches upstream responses there. The client is
created in `services/persistence.ts`; any error or `SIGINT` calls `onError`
which tears down both connections and `process.exit(5)`.

**Error handling** — throw `AppError` from `services/error.ts`
(`AppError.badRequest`, `notFound`, `conflict`, `unauthorized`, `forbidden`,
`tooManyRequests`, `internal`, `serviceUnavailable`). The global
`system.errorHandler` responds `{ identifier, message }` with the matching
status; non-AppError exceptions log and return 500 `API_ERROR`.

## Database

- `node-pg-migrate` v8, config in `.pgmigraterc.json`
  (`migrations-table: pgmigrations`, `ssl: true`, `reject-unauthorized: true`,
  CA injected at runtime via `NODE_EXTRA_CA_CERTS` in `docker-entrypoint.sh`).
- Migrations are plain JS in `./migrations/`, prefixed with a millisecond
  timestamp + sequence number (e.g. `1778200000000_039_comment_restore_audit.js`).
  Create new ones with `make migrate-create name=...` so the prefix is fresh.
- 15 migrations currently in tree; latest is `_039_comment_restore_audit`.
- `pg` Pool is the single connection point (`services/persistence.ts`,
  exported as `database`). Services run plain SQL — no ORM, no query builder.
- Transactions use the `withTx` pattern: `connect()` -> `BEGIN` -> work ->
  `COMMIT`/`ROLLBACK` in `finally`-released client. See
  `CommentService.withTx` in `src/services/comment.ts` for the canonical
  shape; `userService.anonymizeUser` and `exportUserData` also use it.
- Advisory locks gate races where SQL alone cannot: `CommentService.create`
  uses `pg_advisory_xact_lock(hashtext(userId), hashtext(targetId))` to make
  the 30-second cooldown check + insert atomic per (user, target).
- Column convention: snake_case in SQL, aliased to camelCase in `SELECT` so
  query rows match the TS interfaces in `src/models/index.ts` directly.
  Two aliased column constants per table are common (one bare, one `c.`-
  prefixed for joins) — see `COMMENT_COLUMNS_PLAIN` / `_ALIASED`.

## Conventions

- TS is `strict`, target ES2024, module `node20`, `moduleResolution: node16`.
  All intra-`src` imports use the `.js` extension (ESM-style); keep that.
- ESLint flat config (`eslint.config.mjs`): `@typescript-eslint/recommended`,
  `argsIgnorePattern: '^_'`, `no-console: 'off'`. Migrations are ignored.
- File naming: kebab-case (`print-job.ts`, `clip-admin.ts`); one
  controller/service per resource; barrel export from
  `src/services/index.ts` and `src/routes/index.ts`.
- Controllers wrap their bodies in `try { ... } catch (err) { return next(err); }`
  and use small helpers like `requireUser(req)` and `assertUuid(value, field)`
  (see `controllers/comment.ts`). UUIDs are validated client-side of SQL with
  a regex before being interpolated into `::uuid`-cast queries.
- Response shape: success returns the resource JSON directly; errors return
  `{ identifier, message }` (see `system.errorHandler`).
- Comments and user-facing error messages are German in newer code (clips,
  comments, print-request) and English in older code (auth, blog). Match the
  surrounding file; don't translate.

## Gotchas

- **JWT vs `deleted_at`**: `authHandler` does NOT re-check `user.deleted_at`
  per request — JWTs stay stateless. After self-anonymisation the cookie is
  cleared and Twitch tokens revoked, but an already-issued JWT remains valid
  for up to its TTL (~24h). `controllers/user.getMe` does the explicit DB
  lookup and 401+clears the cookie when `deleted_at` is set, so the SPA boots
  the user out on its next refresh. Accepted trade-off; documented at length
  in `services/system.ts`.
- **ADMIN_EMAILS seeding**: `bootstrap.seedAdminPermissions()` runs once
  after `listen()` (best-effort, logs on failure). But if the admin user
  does not exist yet at boot, they get nothing — `bootstrap.seedAdminForUser`
  is therefore also called on every successful login (see commit `31bcd70`).
  When adding a new admin-only permission, add it to `ADMIN_PERMISSIONS` in
  `services/bootstrap.ts` so existing admins pick it up on next login/boot.
- **Comment cooldown is locked, not just queried**: any change to
  `CommentService.create` must keep the `pg_advisory_xact_lock` BEFORE the
  count query. Removing the lock re-opens the double-submit race that
  reviewer feedback closed (review #5). Mods get `bypassCooldown` and take
  the non-tx fast path.
- **`anonymizeUser` is one transaction by design**: seven DELETEs + the PII
  wipe UPDATE all-or-nothing. The external Twitch revoke runs BEFORE
  `BEGIN` so an HTTP call never blocks an open transaction. Adding new PII
  tables? Add the `DELETE` inside the same `withTx`, not after.
- **Self-deletes cannot be restored** (`CommentService.restore`): a `NULL`
  `deletion_reason` marks an author self-delete; mods get a 403 with
  `CANNOT_RESTORE_SELF_DELETE`. Preserves author intent / "Recht auf
  Vergessen". Restore archives the old reason into `last_deletion_reason`
  and stamps `restored_at` / `restored_by_user_id` (audit trail, review #7).
- **OAuth `redirect_uri` is server-side only**: both Twitch and GitHub
  routes build the callback from `config.webUrl` (env-driven), never from
  the request — `Referer` / client-supplied `redirect_uri` was an
  open-redirect vector. Mirrored in `services/config.ts` (`webUrl`).
- **CSRF cookie is double-submit, NOT signed**: `XSRF-TOKEN` is
  `httpOnly: false` on purpose so the SPA can echo it in `X-CSRF-Token`.
  The literal strings `'XSRF-TOKEN'` and the cookie/header comparison in
  `middleware/csrf.ts` are kept inline so CodeQL's
  `js/missing-token-validation` rule recognises the protection.
- **CORS `origin` MUST be a function, not `true` or `'*'`**: credentials
  are enabled. Touching the `cors(...)` block requires re-reading the
  inline comment in `app.ts` — CodeQL `js/cors-permissive-configuration`
  rejects looser configs.
- **OAuth email backfill is one-way**: `setEmailIfMissing` only writes when
  the existing column is NULL. Providers report inconsistent addresses;
  silent overwrites break account recovery.
- **`/agent/*` uses `X-Agent-Token`, not JWT** — agent routes auth inside
  their sub-router; do not wrap them in `system.authHandler`.
- **`/blog/rss.xml` MUST register before `router.use('/blog', blog)`** in
  `routes/index.ts`, else the blog router treats `rss.xml` as a post slug.
